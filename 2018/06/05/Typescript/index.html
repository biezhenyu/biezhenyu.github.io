Typescript

发表于 2018-06-05 | 分类于 typescript


由于使用angular6的技术站进行前端开发，angular是使用Typescript进行开发的，由于水平有限，看起来Typescript文档有点生涩，所以做了一部分的笔记。

基础类型

枚举

  • 使用枚举我们可以定义一些带名字的常量。 使用枚举可以清晰地表达意图或创建一组有区别的用例。 TypeScript支持数字的和基于字符串的枚举。
  • 数字枚举，如果我们定义了一个数字枚举在没有指定初始化的情况下，从0开始，依次加1递增。
  • 异构枚举则是，枚举可以混合字符串和数字成员。但是一般不会这样做。枚举可以指定表达式，会在后面介绍到。
    数字枚举

字符串枚举(字符串枚举没有自增长的行为)

Any

Any指定的是通用的类型, 在非万不得已的情况下, 不要使用Any类型。

Void

Any指定的是通用的类型, 在非万不得已的情况下, 不要使用Any类型。

Null和Undefined

Null Undefined可以赋值给任何类型的变量

Never

表示异常的返回值, 返回never的函数，拥有不会到达终点

类型断言

有时候你会比TypeScript更了解某个值的详细信息, 我们可以通过类型断言指定变量的类型。
类型断言好比其它语言里的类型转换，但是不进行特殊的数据检查和解构。

类型断言的两种写法

interface

接口的作用是为值, 参数, 类, 函数等进行类型检查。

可选属性 ?

interface中定义的属性, 并不完全都是必须的。可以使用?标记为可选属性。定义可选属性后, 可以不传递使用该属性, 但是不能使用interface中没有定义的属性

只读属性 readonly

只读数组 ReadonlyArray

函数类型

interface可以用来描述函数类型, 需要指定参数的类型和返回值的类型

对于函数类型的类型检查来说，函数的参数名不需要与接口里定义的名字相匹配
同样

可索引类型

对于一些未知key值的属性，我们该如何在接口中定义它呢?答案是可以使用字符串索引类型。可索引分为字符串索引和数字索引。但是数字索引是等同于字符串索引的，因为对于key值, 100将会转换为”100”

类接口

interface可以强制一个类实现某一种规范。但是interface只定义类的实例的部分，没有定义类的静态的部分

接口继承

接口与接口之间可以相互继承, 可以达到复用接口的目的

接口继承类

interface可以继承class, interface会继承类的成员的定义(实例的属性以及实例的方法), 但是不包含实现。接口可以继承private和protected的成员，如果接口继承了private和protected的成员，那么这个接口只能被这个类以及它的子类所实现。因为private和protected的成员是私有的，private只能在定义的类的内部访问，protected可以在子类中访问。

class

部分与es6内容重合

public

typescript默认包含了public的修饰符, public修饰符可以自由的访问类的属性以及方法

private

当类的成员被标记为private, 被标记为private的成员将不能在类的外部访问，包括了派生类。

  • TS是基于结构进行类型兼容的, 但是如果成员包含了private或者protected的修饰符, 即使两个类的结构一样，但是protected或者private的成员不来自同一个类型, 那么两个成员之间也是不兼容的（具体请看类型兼容这一章节)

protected

与private类似，但是区别在于protected的成员，虽然在类的外部无法访问，但是可以在派生类中访问父类的protected的成员。构造函数也可以被标记成protected。这意味着这个类不能在包含它的类外被实例化，但是能被继承。可以在派生类中实例化(super)

readonly

可以设置类的属性，为只读属性

构造函数中使用修饰符参数

在构造函数中使用修饰符参数, 相当于把类的声明与赋值放在了同一个地方

getter, setter

和es6一样

静态属性

在es6中，并不支持静态方法, 但是在typescript中实现类的静态方法。通过[类名].的方式，访问静态属性和静态方法

类的类型

定义一个Foo类型, Foo类的实例的类型是Foo, 那么Foo本身的类型是什么呢? 答案是可以用”typeof Foo”获取类的类型

函数

定义函数类型

可选参数，默认参数

可选参数使用, “?”标记为可选参数, 可选择参数的后面必须没有必选参数, 可选参数必须放在所有参数的最后面。默认参数可以直接指定”=”设置为默认参数，带默认值的可选参数，可以不放在参数list的最后面，在使用的时候传入undefined就可以使用默认参数的默认值。

剩余参数

同ES6一样，未知长度的参数可以使用rest参数。

this

箭头函数与this

箭头函数的this的值，是创建时this的值

this参数

this参数，是假参数，位于参数列表的最前面。用来指定函数内部this的类型。因为在字面量对象，函数中的this类型为any, 而非字面量对象

重载

有的时候需要为函数传入各种不同类型的参数，以及不同类型的返回值, 根据不同的参数和返回值需要定义不同的参数声明

泛型

为什么需要使用泛型？

代码需要考虑到可重用性，可以使用泛型，支持多种数据类型

假设遇到以下这种状况, 我们通过的传入和传出值要是类型统一的。如果使用不同类型重载显然是不现实的。如果使用的any, 则会丢失信息。(返回值any, 传入值any，意味着传入是number, 传出是string也是可以的)此时我们可以使用类型变量

泛型变量

类型变量T, 可以是任何类型, 书写泛型函数的时候，请务必考虑到这一点 💭

泛型类

与interface类似, 同样可以指定类型变量参数。可以确认指定的属性使用同样类型的参数

泛型类型

使用interface创建泛型的约束条件

泛型与类类型

使用泛型的类型变量T, 指定类的构造函数的类型(构造函数的返回值)
