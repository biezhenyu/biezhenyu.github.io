vue-cli3实现分环境打包

发表于 2018-08-24 | 分类于 vue ， vue-cli3, 打包


遇到的问题

在vue-cli3的项目中，默认的package.json配置

  • npm run serve时会把process.env.NODE_ENV设置为‘development’
  • npm run build 时会把process.env.NODE_ENV设置为‘production’

此时只要根据process.env.NODE_ENV设置不同请求url就可以很简单的区分出本地和线上环境。

但是实际真实的项目中，会区分预发布，测试版本这些。然而vue-cli3号称0配置，无法直接修改打包文件，那么怎么区分不同的线上环境呢？ 也就是说npm run build时怎么手动更改process.env.NODE_ENV？或者配置一个专门的打包命令。

步骤

修改package.json文件

package.json添加 “build_t”: “vue-cli-service build –mode build_t” （这个是我配置的测试环境）

在项目根目录添加文件.env.build_t
